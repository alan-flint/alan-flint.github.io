Introduction:¶

Data scientists (including budding data scientistics like myself) often find modeling to be the most fun and rewarding part of the data science process. But gaining insights into the model's features, either for business intelligence or model improvement purposes, is an overlooked, yet critical step. Feature importance is the process of figuring out which features have the most predictive power (i.e. most importance) for your model.

In this report, I will outline several feature importance techniques, visualize these strategies, and investigate the variance and statistical significance of the scores. I will be using the popular Boston Dataset in my investigations. The target variable is the median value of homes in $1000's.

Feature Importances only from the X, y Relationship¶

One type of feature importances only uses the relationship between X and y to infer importance of a variable. The first is Spearman's Rank Correlation.

Spearman's Rank Correlation Coefficient¶

This method measures the rank correlation between each individual X variable and the target y. You first separately rank the values in x and y (there are many ways to deal with ties--I used the minimum) and then calculate Pearson's correlation coefficient on the ranked variables. This technique measures the monotonic relationship between x and y and is not influenced by outliers. A problem with this method is that it measures single feature relevance, and if two features, x1 and x2, are highly correlated then they will have similar Spearman's importances scores, even thought only one may actually be important in the model.

Here is the results of Spearman's Rank on the Boston dataset.

Minimal-Redundacy-Maximal-Relevance (mRMR)¶

Another technique is to measure the relevance of a variable, relationship between x and y, and the redundancy of the variable, relationship between x and the other x's, and compute the mRMR. The formula for this is:

$$J_{RMR}(x_{k}) = I(x_{k},y) - \frac{1}{|S|}\sum_{x_{j}\in S} I(x_{k}, x_{j})$$

The function $I(x_{k}, \theta)$ is any measaure of importance, but I will use Spearman's Rank for simplicity. The idea is to penalize the relevance with the average redundancy. mRMR can also be used as a feature selection algorithm, where you add the highest mRMR feature (where S is the current feature space) each iteration. This method is good for parsing out the individual effect of a feature on the target.

Here are the mRMR importances on the Boston Data. Note that it is possible to get negative importances, meaning the variable is more redundant that relevant.

Feature Importances using a Model¶

The other type of feature importance uses the results of a model--calculate the drop in metric score once a feature is removed (drop column) or scrambled (permutation).

Drop Column Importance¶

The idea of drop column importance is to measure the difference in metric score from the full model and a model with a feature dropped. First, train a model on all the features and get a baseline metric score from the validation set, then iteratively, drop a column, re-train the model without that column and get a metric score on the validation set. The difference of the baseline score to each dropped column metric score is the importance measure of that column. However, this procedure is computationally expensive because you have to train the model n_features + 1 number of times.

Here are the results of drop column importance on the Boston data.

Permutation Importance¶

Permutation importance is similar to drop column importance that it iteratively 'drops' a column and re-computes the metric score compared to the baseline. However, permutation importance solves the computationally expensive problem by simply randomly permuting a column then re-computing the metric score. The model is only trained once for the baseline, and the column drop is simulated by randomizing the column. Randomly shuffling the column breaks all relationships and essentially makes that column's predictive power zero.

Here are the results of permutation importance for Boston.

Measures of Variance and Statistical Significance¶

Variance¶

Once we have measures of feature importance, one should consider how the results change when repeated over and over again. If we consider our dataset a sample of the actual population dataset, computing the feature importances on a dataset is the same as computing a parameter for one sample of a population. From the Law of Large numbers, we know that as we repeat an experiment many times, the average of the parameter will trend towards the expected value of, or population, paramter. In our case, the parameter is feature importances. To simulate getting a different sample of data, we can bootstrap (sample n rows with replacement).

In my example, I bootstrapped the Boston data 250 times, and computed the mRMR importances for each sample. The bar plot below shows the 'true' feature importance and the spread, or variance, of the importances. The error bars denote $\pm1$ standard deviation from the mean.

Statistical Significance¶

The ability to compute the variance of the importances is great, but can we actually measure if a feature is statistically significant? The answer is yes. If we can create a null distribution for each feature importance, then we can empirically compute a p-value (probability the feature importance is from the null distribution) for each feature. The process goes like this,

  1. Compute the real, baseline, feature importances using the normal X, y dataset
  2. Iteratively for n_trials (create the null distribution):
    1. Shuffle the y values
    2. Compute feature importances using X and shuffled y
    3. Compare these importances to the baseline
  3. Count the number of times the importances on the shuffled data exceed the baseline importances for each feature
  4. This count/n_trials is the empirical p-value for each feature

If the p-value is greater than some threshold (usually 5%), then that feature is not statistsically significant.

In my example below, I used permutation importance as my importance measure.

In [62]:
baseline, importances, p_val = empirical_p_value(rf, X_train, X_val, y_train, y_val, 500)

Conclusion:¶

A data scientists job is not over once they fit a good model. Some of the most crucial insights can come from a post-hoc analysis of the model and it's features. The methods described in this report, like Spearman's Rank correlation, permutation importance, and statistical significance, can help one explain how and why a model predicts the way it does. Those explanations are critical for business intelligence and can help facilitate discussions to further improve the model.. which is the fun part, right?
